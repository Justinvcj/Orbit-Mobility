## Core System Architectures

### 1. State Rehydration & Lifecycle Fluidity (Phase 1)
Mobile operating systems aggressively suspend background applications to save memory. Equinox counteracts this via a bipartite rehydration model:
- **Client Native Observers:** Both apps implement `WidgetsBindingObserver` to detect `AppLifecycleState.resumed`.
- **Stateless Reconnects:** Upon foregrounding, the client makes a fast-path HTTP GET request to `/api/active-ride/:phone`.
- **Ledger Source of Truth:** The server responds with the currently active ride schema from the Supabase database, allowing the client to instantly rebuild `rideId`, `status`, and `fare` memory clusters without waiting for a fresh Socket handshake.
- **Socket Ack Pipeline:** Critical payload emissions (e.g., `accept_ride`) mandate server-side `.json` un-packing acknowledgments (`ack(response)`). This prevents client-side UI deadlocks if network packets drop over cellular networks.

### 2. Telemetry & Geospatial WebSockets
The Node.js server acts as an ephemeral routing switchboard rather than just an API:
- **Transient Memory Ledger:** Phone numbers act as primary keys. `register_rider` and `register_driver` events cast the ephemeral Socket ID to the `phone` string, allowing O(1) bipartite targeting without DB writes.
- **Simulated Movement Stream:** Until hardware GPS is implemented, `simulateMovement()` mechanically ticks every 2 seconds on the server. It calculates distance subsets between `pLat/pLng` and `dLat/dLng` and sprays `driver_location_update` events back to the bounded Rider socket.
- **Rotational Interpolation:** The Driver car dynamically calculates compass bearing utilizing `math.atan2`. The Flutter UI hooks this raw rad bearing into `flutter_animate` engines, creating smooth physical turning animations rather than jerky 90-degree instant snaps.

### 3. Asymmetric Security & Validation Engine (Phase 2)
To prevent rogue pick-ups, authorization is split across the clients:
- **Server Injection:** When a driver accepts a ride (`accept_ride`), the server generates a cryptographically random 4-digit OTP. It injects the initial ride record into Supabase `rides` table alongside this OTP.
- **Rider Vision:** The OTP is pushed *only* to the Rider’s UI via the `ride_accepted` socket payload.
- **Driver Gate:** The driver cannot transition the vehicle state to `IN_PROGRESS` (which starts billing) until they open an `AlertDialog`, physically solicit the 4-digit code from the rider out-loud, and match it against the ephemeral state payload exactly.

### 4. High-Performance GIS Subsystems (Phase 3)
Equinox aggressively optimizes location mapping to prevent device thermal throttling and API rate limiting:
- **Anti-Burst Search (Debouncing):** Photon REST geographic searches are guarded by a 500ms `Timer` throttle. Typing "Bangalore" triggers only 1 API call after keystrokes cease, not 9 sequential queries.
- **Hyper-Local Bounding Boxes:** Forward geocoding is surgically weighted. By utilizing the `geolocator` plugin, the search query calculates a `~50KM` mathematical bounding box (`minLon,minLat,maxLon,maxLat`) offset around the device's physical hardware coordinates. This ensures "Main Street" returns the local road, not a road in another country.
- **Tile Architecture:** Standard Google/Apple map wrappers were bypassed for highly performant, raw `flutter_map` TileLayers. We stream `CartoDB Dark Matter` retina-scaled PNG grids (`{r}`) directly to the canvas, allowing unrestricted visual overriding for the Midnight Premium aesthetic.

### 5. "Midnight Premium" UI/UX Core (Phase 4)
The entire surface layer operates on strict OLED-optimized design tokens:
- **Zero-Bleed OLEDs:** Scaffolds forced to `#0A0A0A` allow AMOLED screens to completely shut off background pixels, giving the illusion that the UI interacts with the physical phone bezel.
- **Frosted Glassmorphism:** All temporal elements (BottomSheets, Dialogs, AppBars) are detached from the base structure using `ClipRRect(BackdropFilter(sigmaX: 10, sigmaY: 10))`. Surfaces use translucent `0.05` opacity white with `0.1` opacity structural bounding borders to hover distinctly over map tiles.
- **Haptic Neural Feedback:** Over 5 critical user action states (Ride Accept, Arrival, verification) are hooked into `Vibration.vibrate(duration: 50)`. This micro-burst creates visceral physical feedback complementing visual AnimatedSwitchers, giving the software a premium, native weight.
