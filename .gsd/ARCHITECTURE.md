## Data Flow

1. **Client Registration:** Both Rider and Driver authenticate implicitly via phone number. `socket.emit('register_rider' / 'register_driver')` binds Socket ID to user identity.
2. **Ride Handshake:** 
   - Rider requests ride (HTTP POST to Supabase + Socket ping to Backend).
   - Backend dispatches to active drivers via `new_ride_request`.
   - First Driver to accept locks the ride in. Server generates a dynamic OTP and syncs state (`ride_accepted`) to both clients.
3. **Trip Lifecycle:** 
   - Driver marks `ARRIVED`.
   - Rider provides OTP to Driver.
   - Driver submits OTP via Socket payload. On successful match, Backend flips state to `IN_PROGRESS` and starts driver telemetry simulation.
   - Driver completes ride; server triggers `/api/update-wallet` transaction.
